Contribute

Stencila is an open-source, community-driven project. We encourage and welcome contributions from all community members. There are a number of ways you can contribute. For example, you can help:

  • shape Stencila's direction and features ,
  • improve this website,
  • develop Stencila software.

Help shape Stencila

Help shape the Stencila's direction and features. Join our Community Forum to get the latest news. Chat with the team on our Gitter channel, on Twitter @Stencila, or email to hello@stenci.la.

Help improve this website

Did you spot a typo or other mistake on this website? Do you have a suggestion for a better documentation? Would you like to write a blog post for Stencila?

Just click on the Edit link at the bottom of the page you want to improve! See the the Stencila Website reposiroy for more detailed instructions.

Help develop Stencila software

If you develop code for data manipulation (in Python, R, and other languages), you may be interested in:

  • Core library
  • Domain-specific libraries

If you are a Javascript developer, you may be interested in helping out with:

  • Stencila user interfaces
  • Stencila for Node.js
  • Stencila Desktop
  • Stencila CLI (Command Line Interface)

If you are a Python developer, you may be interested in helping out with:

  • Stencila for Python
  • Stencila Hub

If you are a R developer, you may be interested in helping out with:

  • Stencila for R

If you are into cloud computing, Docker and Kubernetes, you may be interested in:

  • Stencila Cloud

If you build Docker containers, contribute to:

  • Stencila Images

General contribution guidelines

We prefer the contributions to be made via GitHub.

  1. If you do not have a GitHub account, you can send us comments by email. However, we will be able to respond more quickly if you use one of the other methods described below.

  2. If you have a GitHub account, or are willing to create one, but do not know how to use Git, you can report problems or suggest improvements by creating an issue. This allows us to assign the item to someone and to respond to it in a threaded discussion.

  3. If you are comfortable with Git, you can submit a pull request (PR). In Stencila we follow a commonly used workflow for contributing to open source projects (see also GitHub instructions).

By contributing, you agree that we may redistribute your work under our license. Everyone involved with Stencila agrees to abide by our code of conduct.

These contribution guidelines are based on the guidelines provided by Software and Data Carpentry.
